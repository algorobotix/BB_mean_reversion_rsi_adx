# Руководство пользователя: trading_notebook.ipynb

## Что это такое

`trading_notebook.ipynb` — интерактивный Jupyter-ноутбук для анализа, бэктеста и оптимизации торговой стратегии **BB Mean Reversion** на криптовалютных фьючерсах Binance.

**Стратегия работает на принципе возврата к среднему:**
- Рынок временно уходит за пределы полос Боллинджера при перекупленности/перепроданности (RSI)
- Вход в сделку происходит только на боковом рынке (ADX ниже порога), когда вероятность возврата к средней полосе высока
- Выход — когда цена достигает средней полосы BB или срабатывает стоп-лосс по ATR

## Требования

### Программное обеспечение
- **Python 3.10+**
- **Jupyter Notebook** или **JupyterLab** (или VS Code с расширением Jupyter)

### Установка зависимостей
```bash
pip install -r requirements.txt
```

> Если это первый запуск — выполните **Ячейку 0** (установка) прямо в ноутбуке: она автоматически поставит `nest_asyncio`, `ipywidgets` и `scikit-learn`.

### Интернет-соединение
Ячейки 2 и 3 обращаются к внешним API:
- **CoinGecko** — для получения списка топ-монет по капитализации
- **Binance Futures API** — для загрузки исторических свечей (ключи API не нужны)

## Архитектура ноутбука

Ноутбук состоит из **5 последовательных ячеек** (плюс одна установочная). Каждая ячейка самодостаточна, но опирается на переменные, заданные в предыдущих.

```
[Ячейка 0] Установка пакетов  (один раз)
     ↓
[Ячейка 1] ⚙️  Настройки       ← единственная точка конфигурации
     ↓
[Ячейка 2] 🔍  Список монет    ← CoinGecko + Binance Futures фильтр
     ↓
[Ячейка 3] 📥  Загрузка свечей ← Binance API → CSV в klines/
     ↓
[Ячейка 4] 📊  Бэктест         ← метрики + интерактивный график
     ↓
[Ячейка 5] 🔬  Оптимизация     ← Optuna TPE/CMA-ES + Walk-Forward
```

## Подробное описание ячеек

### Ячейка 1 — Настройки

Единственное место, где нужно менять параметры. Все остальные ячейки только читают переменные `_NB_*`.

#### Режим работы

| Переменная | Значение | Описание |
|---|---|---|
| `_NB_SYMBOL` | `'BTCUSDT'` | Анализ одной монеты |
| `_NB_SYMBOL` | `None` | Режим топ-N (несколько монет) |
| `_NB_TOP_N` | `5` | Сколько монет взять из топа (при `_NB_SYMBOL=None`) |

#### Период и таймфрейм

| Переменная | Пример | Описание |
|---|---|---|
| `_NB_INTERVAL` | `'1h'` | Таймфрейм: `1m`, `5m`, `15m`, `30m`, `1h`, `4h`, `1d` |
| `_NB_START_DATE` | `'2025-01-01'` | Начало периода (включительно) |
| `_NB_END_DATE` | `'2025-12-31'` | Конец периода (включительно) |
| `_NB_KLINES_DIR` | `'klines'` | Папка для CSV-файлов |

> **Рекомендация по периоду:** для надёжных результатов используйте минимум 6–12 месяцев исторических данных.  
> На `1h` это ~4 000–9 000 свечей, на `15m` — ~16 000–35 000.

#### Параметры стратегии

| Параметр | Диапазон | Описание |
|---|---|---|
| `bb_len` | 10–50 | Период Bollinger Bands. Меньше = чувствительнее полосы |
| `bb_std` | 1.5–3.0 | Ширина полос (множитель σ). Больше = реже сигналы, шире полосы |
| `rsi_len` | 7–21 | Период RSI. Стандарт — 14 |
| `rsi_lower` | 20–35 | Порог перепроданности для LONG. Ниже = строже фильтр |
| `rsi_upper` | 65–80 | Порог перекупленности для SHORT. Выше = строже фильтр |
| `adx_len` | 7–21 | Период ADX. Стандарт — 14 |
| `adx_threshold` | 20–35 | Максимальный ADX для входа. Ниже = только сильный боковик |
| `sl_atr_mult` | 1.0–5.0 | Ширина стоп-лосса в ATR. Больше = шире стоп |
| `trade_usdt` | 50–500 | Размер позиции в USDT на сделку |

> **Логика входа (LONG):** Close < BB_lower **И** RSI < `rsi_lower` **И** ADX < `adx_threshold`  
> **Логика входа (SHORT):** Close > BB_upper **И** RSI > `rsi_upper` **И** ADX < `adx_threshold`  
> **Выход:** цена достигла средней полосы BB (`BB_MID`) или ATR-стоп

#### Параметры оптимизации

| Параметр | Рекомендация | Описание |
|---|---|---|
| `_NB_N_TRIALS` | 100–500 | Число попыток Optuna. Больше = точнее, но дольше |
| `_NB_N_JOBS` | 1 | Параллельных процессов. На Windows оставьте 1 |
| `_NB_OBJECTIVE` | `'composite'` | Целевая функция оптимизатора (см. ниже) |
| `_NB_MIN_TRADES` | 10–20 | Минимум сделок, иначе trial отклоняется |
| `_NB_OUT_OF_SAMPLE` | 0.20 | Доля данных для проверки (OOS). Обычно 20–30% |
| `_NB_WALK_FORWARD` | `True` | Walk-forward валидация для проверки устойчивости |
| `_NB_WF_FOLDS` | 3–10 | Число фолдов walk-forward |
| `_NB_SAMPLER` | `'tpe'` | Алгоритм поиска Optuna (см. ниже) |
| `_NB_OPT_SYMBOL` | `None` | Символ для оптимизации (None = берётся из _NB_SYMBOL) |

**Целевые функции (`_NB_OBJECTIVE`):**

| Значение | Описание |
|---|---|
| `composite` | Взвешенная сумма Sharpe + Sortino + Calmar + PF + SQN. **Лучший выбор по умолчанию** |
| `sharpe` | Максимизирует соотношение доходность/риск |
| `sortino` | Аналог Sharpe, но учитывает только отрицательную волатильность |
| `calmar` | Доходность / макс. просадка |
| `risk_adjusted` | Доходность с жёстким штрафом за просадку |

**Алгоритмы (`_NB_SAMPLER`):**

| Значение | Описание |
|---|---|
| `tpe` | Tree-structured Parzen Estimator — байесовский поиск. **Лучший для большинства задач** |
| `cmaes` | Эволюционный алгоритм. Хорошо работает на непрерывных параметрах |
| `nsga2` | Многоцелевая оптимизация (экспериментально) |
| `random` | Случайный перебор. Полезен для первичной разведки |

### Ячейка 4 — Бэктест

Запускает стратегию через Backtrader. Поведение зависит от числа монет:

**Одна монета:**
- Транспонированная таблица с полным набором метрик
- Интерактивный Plotly-график в стиле TradingView: свечи, полосы BB, RSI, ADX, точки входа/выхода, кривая капитала

**Несколько монет:**
- Сводная таблица, отсортированная по Sharpe
- Цветовой градиент: зелёный = хорошо, красный = плохо (Sharpe, Return, WinRate, MaxDD)

**Ключевые метрики:**

| Метрика | Хорошо | Описание |
|---|---|---|
| Sharpe | > 1.0 | Доходность / волатильность (годовая) |
| Sortino | > 1.5 | Аналог Sharpe, только отриц. волатильность |
| Profit Factor | > 1.3 | Сумма прибылей / сумма убытков |
| Max DD % | < 15% | Максимальная просадка от пика |
| Win Rate % | > 45% | Процент прибыльных сделок |
| SQN | > 1.0 | System Quality Number — качество системы |

---

### Ячейка 5 — Оптимизация параметров

Наиболее мощная ячейка. Использует Optuna для поиска оптимальных параметров стратегии.

**Что происходит внутри:**

1. **Разбивка данных:** исторический период делится на In-Sample (train, 80%) и Out-of-Sample (OOS, 20%)
2. **Оптимизация на train:** Optuna перебирает `_NB_N_TRIALS` комбинаций параметров, отбрасывая trials с менее чем `_NB_MIN_TRADES` сделками
3. **Проверка на OOS:** лучшие параметры прогоняются на данных, которые оптимизатор не видел
4. **Walk-Forward валидация** (если `_NB_WALK_FORWARD=True`): весь датасет разбивается на `_NB_WF_FOLDS` скользящих окон, параметры тестируются на каждом тестовом сегменте
5. **Детекция переобучения:** автоматическая проверка деградации Sharpe, Win Rate и Return между IS и OOS

**Вердикты переобучения:**

| Вердикт | Смысл |
|---|---|
| `OK` | Параметры устойчивы, переобучения не обнаружено |
| `BORDERLINE` | Лёгкая деградация — использовать с осторожностью |
| `OVERFIT` | Параметры подогнаны под историю и не работают на новых данных |

**Графики Optuna:**
- **Optimization History** — как улучшался результат от trial к trial
- **Param Importances** — какие параметры сильнее всего влияют на результат
- **Parallel Coordinate** — визуализация связей между параметрами и целевой функцией
- **Walk-Forward** — Return % и Sharpe по каждому фолду

> После оптимизации лучшие параметры доступны в переменной `_NB_BEST_PARAMS`. Скопируйте их в `_NB_STRATEGY_PARAMS` в ячейке 1 и запустите ячейку 4 для итогового бэктеста.

## Советы и рекомендации

### По выбору периода
- Используйте **минимум 1–2 года** данных для надёжных выводов
- Чем короче таймфрейм (`15m`, `1h`), тем больше свечей и тем дольше оптимизация
- Выбирайте период, охватывающий разные рыночные режимы: бычий тренд, медвежий тренд, боковик

### По оптимизации
- Начните с `_NB_N_TRIALS = 50–100` для проверки корректности настроек
- Для финального поиска используйте `300–1000` trials
- `_NB_MIN_TRADES = 10–20` защищает от «счастливых» параметров с 1–2 сделками
- Вердикт `OVERFIT` при walk-forward — сигнал увеличить `_NB_MIN_TRADES` или расширить период данных
- `_NB_OUT_OF_SAMPLE = 0.20–0.30` — стандартный диапазон; меньше = больше данных для train, но слабее проверка

### По интерпретации результатов
- Ориентируйтесь в первую очередь на **Sharpe > 1.0** и **Profit Factor > 1.3**
- Высокий Win Rate (> 60%) при отрицательном PF — признак плохого соотношения риск/прибыль
- Малое число сделок (< 20 за год) делает метрики статистически ненадёжными
- **Никогда** не торгуйте реальными деньгами по параметрам с вердиктом `OVERFIT`

### По производительности
- На 1h-данных 100 trials занимают ~2–5 минут
- На 15m-данных 100 trials могут занять 10–20 минут
- `_NB_N_JOBS = 1` — безопасный режим для Windows; увеличение может привести к зависанию Jupyter

## Часто задаваемые вопросы

**Q: Ноутбук выдаёт `TqdmWarning: IProgress not found`**  
A: Выполните ячейку 0 установки пакетов — она поставит `ipywidgets`. После этого перезапустите ядро Jupyter (Kernel → Restart).

**Q: Optuna не строит график важности параметров (`No module named 'sklearn'`)**  
A: Выполните ячейку 0 — она поставит `scikit-learn`. После перезапуска ядра графики будут работать.

**Q: Ячейка 2 зависает или возвращает пустой список**  
A: Проверьте интернет-соединение. CoinGecko иногда временно недоступен — повторите через 1–2 минуты. Или задайте `_NB_SYMBOL = 'BTCUSDT'` для обхода CoinGecko.

**Q: Все trials в оптимизации pruned**  
A: Уменьшите `_NB_MIN_TRADES` до 5 или увеличьте период данных. При коротком периоде и большом `_NB_MIN_TRADES` оптимизатору не хватает сделок для оценки.

**Q: Walk-forward показывает разные результаты каждый раз**  
A: Это нормально — Optuna использует случайную инициализацию. Для воспроизводимости можно зафиксировать seed в коде ячейки 5 (`sampler = optuna.samplers.TPESampler(seed=42)`).

**Q: Бэктест показывает отличные результаты, но на реальной торговле стратегия убыточна**  
A: Наиболее вероятные причины — переобучение (вердикт `OVERFIT`) или слишком короткий тестовый период. Используйте минимум 2 года данных и доверяйте только параметрам с вердиктом `OK` по walk-forward.
